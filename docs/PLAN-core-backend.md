# Implementation Plan: Cuan Core Backend

## Overview

Implement the `core` Elysia.js backend for Cuan with email/password authentication (Better Auth), PostgreSQL persistence (Drizzle ORM), and a protected chat endpoint that uses an external OpenAI-compatible LLM ("OpenModel") to extract expenses from natural language and save them to the database.

## Architecture Decisions

1. **Elysia.js runtime** — Lightweight, Bun-native web framework; mounts Better Auth's standard web handler directly via `app.mount('/api/auth', auth.handler)`.
2. **Better Auth with Drizzle adapter** — Self-hosted email/password auth; session cookies are validated with `auth.api.getSession`. Better Auth's tables are generated by `@better-auth/cli` and managed alongside application tables by `drizzle-kit`.
3. **PostgreSQL + Drizzle ORM** — `pg` pool for connectivity; `drizzle-orm/pg-core` for the `expenses` table.
4. **OpenAI-compatible LLM client** — "OpenModel" is treated as a generic OpenAI-compatible provider configured through env vars (`OPENMODEL_API_KEY`, `OPENMODEL_BASE_URL`, `OPENMODEL_MODEL`). This avoids hard-coding an unknown provider URL while remaining compatible with OpenAI, DeepInfra, Together, Groq, etc.
5. **Env validation with Zod** — Runtime validation of required configuration; fails fast on startup.
6. **Cents in DB, decimal in API** — `amount` stored as `numeric(12,2)` (Drizzle `numeric`) to avoid floating-point rounding. API accepts/returns numeric values.
7. **Test strategy** — Unit tests for expense extraction with a fake LLM client; integration tests for the chat endpoint require a running DB and are gated behind an env variable.

## Risks and Mitigations

| Risk                                                      | Impact | Mitigation                                                                                                 |
| --------------------------------------------------------- | ------ | ---------------------------------------------------------------------------------------------------------- |
| Better Auth CLI-generated schema changes between versions | Medium | Pin `better-auth` and `@better-auth/cli` versions; regenerate migrations in Task 2.                        |
| OpenModel provider details unknown                        | Medium | Externalize base URL, model, and API key to env vars; default to OpenAI-compatible `/v1/chat/completions`. |
| No running PostgreSQL in CI/test                          | Medium | Integration tests check for `TEST_DATABASE_URL` and skip if absent; unit tests use fake client.            |
| Elysia/Better Auth handler type mismatch                  | Low    | Use `app.mount(path, handler)` which accepts a standard `Request => Response` handler.                     |
